## Was wurde gebaut + warum

### ADR-001 (`docs/superpowers/findings/2026-04-23-adr-smart-routing-rollout-gate.md`)

**Problem:** In früherer Session (2026-04-20) landete ein konservativer keyword-heuristic Router (`python-backend/agent/llm/smart_routing.py`) mit Migration 026 als "Stufe-0 MVP", geflaggt als `NEEDS HOLISTIC REVIEW`. 6 offene Design-Fragen. Gated off default; aber: bevor er pro-User angeschaltet werden kann, brauchte es eine adversarial walk-through damit kein silent correctness/compliance/observability bug live geht.

**Was:** `sota-contrarian stakes=high` Review → Written ADR mit Verdict pro Frage + Rollout-Gate (G1-G6 + P1 Inversion-Target). Critical findings waren:
1. **Silent vendor substitution** (User wählt Anthropic → simple turn silent auf OpenAI cheap) — GDPR/DPA-Risiko.
2. **`get_user_smart_routing_config` öffnet DB-Conn per iteration-0 turn für ALLE User**, auch `enabled:false` — Latenz + Pool-Pressure.
3. **Keyword-Liste war English-only** → complex DE query ("analysiere mein Portfolio") routet CHEAP, trivial EN ("please review this") bleibt PRIMARY — heuristic ist für DE-User invers zum behaupteten conservative-bias.
4. **Credential-Pool mismatch** — nach state["model"]=cheap acquire(cheap_provider) returns None wenn User nur primary-provider creds hat → state.api_key (wrong key) durchgereicht → 401.
5. **A/B Harness contamination** — self-selected SQL-enabled cohort confounds `langgraph vs simple_loop` fitness measurement.
6. **Deprecation-Cliff** auf literal `cheap_model: "gpt-4o-mini"` + silent except swallows future 404.

**Files:** `python-backend/agent/llm/smart_routing.py`, `tests/agent/llm/test_smart_routing.py`.

## Lessons / Surprises

- **DE-keyword bias war nicht obvious** aus dem Contrarian-output alleine — erst durch concrete example walk-through ("analysiere mein Portfolio") klar dass die heuristic systematically inverted für den target-user.
- **`state["model"]` mutation im llm_node ist nicht idempotent** — credentialpool.acquire wird NACH der mutation mit cheap_provider aufgerufen. Ohne G2 wäre das ein silent 401 für jeden Anthropic-only User. Classic side-effect-order bug, nur sichtbar wenn man end-to-end walks through.
- **A/B Harness confounding** war der fundamentalste Finding — der "fitness data" für runner-variant-comparison wäre während rollout unbrauchbar gewesen ohne die routing-dimension auf demselben row. Recording the dimension musste BEFORE data collection passieren, nicht post-hoc.
- **AgentExecutionContext ist `frozen=True`** — `dataclasses.replace` war der richtige Weg, nicht mutable ctx oder ein separates thread-local.
- **`podman-compose` + `gitnexus analyze` parallel** — reindex läuft gerade (background task `bui37spi0`), ~3min. Danach sollte G4-Refactor impact sauber sichtbar sein.
