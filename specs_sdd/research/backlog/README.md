## Promotion Rule

Promote one topic to a feature only when there is a concrete implementation
scope and owner. Until then, keep it as research/backlog and link affected
feature specs when a subpiece is relevant.

Examples:

- `exec-transformersjs` can become part of Feature 007 if browser-side title
  generation becomes current work.
- `exec-media-ingestion` can become a feature once document/media ingestion is
  selected as a real implementation slice.
- `exec-ebm` can remain research for Feature 014 unless it becomes a concrete
  harness scoring implementation.
