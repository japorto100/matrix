# Journal Conventions

The journal is a chronological execution log. It does not replace feature
`tasks.md`.

Use journal entries for:

- bootstrap sessions
- migration sessions
- live verify runs
- unexpected findings
- handoff notes

Do not use journal entries as the only place for:

- feature requirements
- binding architecture decisions
- open implementation tasks
- acceptance criteria

If a journal note creates work, link it to a feature task. If it creates a
decision, promote it to an ADR.
